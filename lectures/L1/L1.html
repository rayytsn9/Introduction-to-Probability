Motivating Example - Birthday Problem

Question:

What is the minimum number of people needed in a group for there to be at least a 50% probability that two people share the same birthday?

Solution Approach

Use complement probability: P(at least one shared birthday) = 1 - P(no shared birthdays)

Assumptions

  • 365 days in a year (ignoring leap years)
  • All birthdays are equally likely
  • Birthdays are independent

Calculation

For n people, the probability that no two share a birthday is:

P(no shared) = (365/365) √ó (364/365) √ó (363/365) √ó ... √ó ((365-n+1)/365)

Therefore:

P(at least one shared) = 1 - P(no shared)

Answer

The minimum number of people needed is 23.

At n = 23, the probability of at least one shared birthday is approximately 50.73%.

Key Insight

This result is counterintuitive - most people guess much higher numbers (often 100+). The birthday problem demonstrates how quickly probabilities can accumulate with independent events.

Probability Space (Œ©, F, P)

A probability space is a mathematical construct that provides a formal framework for probability theory. It consists of three components:

‚Ä¢ Sample space (Œ©):

The set of all possible outcomes of a random experiment

  • Example: For a coin flip, Œ© = {H, T}
  • Example: For rolling a die, Œ© = {1, 2, 3, 4, 5, 6}

‚Ä¢ Collection of events (F):

A œÉ-algebra (sigma-algebra) of subsets of Œ©

  • Contains all events we can assign probabilities to
  • Must be closed under complementation and countable unions
  • Example: For coin flip, F = {‚àÖ, {H}, {T}, {H,T}}

‚Ä¢ Probability measure (P):

A function P: F ‚Üí [0,1] that assigns probabilities to events

  • P(‚àÖ) = 0 (impossible event has probability 0)
  • P(Œ©) = 1 (certain event has probability 1)
  • For disjoint events A‚ÇÅ, A‚ÇÇ, ..., P(‚à™·µ¢ A·µ¢) = Œ£·µ¢ P(A·µ¢) (countable additivity)

Kolmogorov's Axioms (1933):

The probability measure P must satisfy these three axioms:

i. Non-negativity:

P(A) ‚â• 0 for all events A ‚àà F

ii. Normalization:

P(Œ©) = 1 (the probability of the sample space is 1)

iii. Countable additivity:

For any countable collection of mutually exclusive / disjoint events A‚ÇÅ, A‚ÇÇ, A‚ÇÉ, ... in F:

P(‚à™·µ¢‚Çå‚ÇÅ^‚àû A·µ¢) = Œ£·µ¢‚Çå‚ÇÅ^‚àû P(A·µ¢)

Key Properties Derived from the Axioms:

  • P(‚àÖ) = 0
  • P(A·∂ú) = 1 - P(A) (complement rule)
  • If A ‚äÜ B, then P(A) ‚â§ P(B) (monotonicity)
  • P(A ‚à™ B) = P(A) + P(B) - P(A ‚à© B) (inclusion-exclusion principle)

Example 1 - Flipping a Fair Coin

Probability space (Œ©, F, P):

‚Ä¢ Sample space: Œ© = {H, T} (Heads, Tails)

‚Ä¢ Collection of events: F = {‚àÖ, {H}, {T}, {H,T}} (all possible subsets)

‚Ä¢ Probability measure: P({H}) = 1/2, P({T}) = 1/2, P(‚àÖ) = 0, P({H,T}) = 1

Kolmogorov's axioms are satisfied since:

  • i. P(A) ‚â• 0 for all A ‚àà F (all probabilities are non-negative)
  • ii. P(Œ©) = P({H,T}) = 1 (probability of sample space is 1)
  • iii. For disjoint events, P(A ‚à™ B) = P(A) + P(B) (e.g., P({H} ‚à™ {T}) = P({H}) + P({T}) = 1/2 + 1/2 = 1)

Example 2 - Rolling a Standard Six-Sided Die

Probability space (Œ©, F, P):

‚Ä¢ Sample space: Œ© = {1, 2, 3, 4, 5, 6}

‚Ä¢ Collection of events: F = all possible subsets of Œ© (2^6 = 64 subsets)

‚Ä¢ Probability measure: P({i}) = 1/6 for each i ‚àà {1, 2, 3, 4, 5, 6}

A possible event in Œ© is:

A = {the outcome is divisible by 3} = {3, 6}

P(A) = P({3, 6}) = P({3}) + P({6}) = 1/6 + 1/6 = 1/3

Example 3 - Flipping a Fair Coin Three Times

Probability space (Œ©, F, P):

‚Ä¢ Sample space: Œ© = {HHH, HHT, HTH, HTT, THH, THT, TTH, TTT}

‚Ä¢ Collection of events: F = all possible subsets of Œ© (2^8 = 256 subsets)

‚Ä¢ Probability measure: P({œâ}) = 1/8 for each outcome œâ ‚àà Œ©

A possible event in Œ© is:

Code Examples: Simulating Probability Spaces

JavaScript Simulation

Here's how we might simulate a coin toss probability space:

// JavaScript simulation of coin toss
function simulateCoinToss(trials = 1000) {
    let heads = 0;
    let tails = 0;
    
    for (let i = 0; i < trials; i++) {
        const result = Math.random() < 0.5 ? 'H' : 'T';
        if (result === 'H') heads++;
        else tails++;
    }
    
    console.log(`Heads: ${heads}, Tails: ${tails}`);
    console.log(`P(H) ‚âà ${heads/trials}, P(T) ‚âà ${tails/trials}`);
    
    return { heads, tails };
}

// Run simulation
const results = simulateCoinToss(10000);

Python Implementation

Here's the same concept in Python with statistical analysis:

import random
import matplotlib.pyplot as plt

def simulate_coin_toss(trials=1000):
    """Simulate coin tosses and calculate probabilities"""
    heads = 0
    tails = 0
    
    for _ in range(trials):
        result = random.choice(['H', 'T'])
        if result == 'H':
            heads += 1
        else:
            tails += 1
    
    # Calculate empirical probabilities
    p_heads = heads / trials
    p_tails = tails / trials
    
    print(f"Trials: {trials}")
    print(f"Heads: {heads} (P ‚âà {p_heads:.4f})")
    print(f"Tails: {tails} (P ‚âà {p_tails:.4f})")
    
    return {'heads': heads, 'tails': tails, 'probabilities': [p_heads, p_tails]}

# Example usage
if __name__ == "__main__":
    results = simulate_coin_toss(10000)
    print(f"Theoretical P(H) = 0.5, Empirical P(H) = {results['probabilities'][0]:.4f}")
